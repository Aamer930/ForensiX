# ForensiX Design Context

## Target Audience
University students and instructors — digital forensics / cybersecurity course context. Demo audience includes professors and peers who judge technical depth and UI polish equally.

## Use Cases
- Upload forensic artifacts (memory dumps, PE executables, PCAP captures, Windows event logs)
- Watch AI agent analyze files in real-time via live terminal stream
- Review structured results: risk score, MITRE ATT&CK heatmap, entropy chart, timeline, threat graph
- Export professional PDF report for submission

## Brand Personality / Tone
**Tactical + Cerebral.** The interface should feel like a professional security operations center — precise, fast, intelligent. Not playful. Not corporate. Think: classified terminal output meets modern data visualization. It should feel like the tool *knows things* the user doesn't yet.

Words: **sharp**, **authoritative**, **forensic**

## Aesthetic Direction
Keep and refine the existing dark + neon-green system. Do NOT redesign from scratch.

### Existing tokens (preserve these)
- Background darkest: `#020617`
- Surface: `#0F172A`
- Surface 2: `#1E293B`
- Accent green: `#22C55E`
- Muted text: `#64748B`
- Secondary text: `#94A3B8`
- Fonts: Fira Code (mono labels/data), Fira Sans (body)
- Custom: `scanlines`, `grid-bg`, `neon-text`, `btn-neon`, `fade-in-up`, `card-hover`, `pulse-glow`
- Neon shadow: `0 0 10px #22C55E, 0 0 30px rgba(34,197,94,0.4)`

### Polish priorities
- Tighten spacing consistency — some sections use ad-hoc padding
- Ensure all interactive states (hover, focus, active, disabled) are complete
- Verify light mode still looks intentional (not an afterthought)
- Make transitions feel snappy and consistent (150–200ms ease-out)
- Typography hierarchy: labels clearly smaller/dimmer than values
- Empty states should feel designed, not blank
- No layout shift on data load

## Quality Bar
Flagship. This is a university capstone demo — it needs to impress professors and peers at first glance and hold up to close inspection.
